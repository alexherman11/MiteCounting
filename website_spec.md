Counting website

General website criteria

- The goal of this website is to replace physical mite click counters.
  It should be easy to use and have good visual feedback.

- When the user enters the website they should be prompted to pick
  between three different modes of counting, SWD, mites, and lygus. On
  click the website should load up those columns.

- Keys on the keyboard to correspond to each category with two modes for
  data entry.

  - Each column in the display is its own species, with the top being
    adult, middle row being nymph, and the bottom being egg.

  - Alternatively, there should be a mode where the user can hit space
    to enter Adult mode, enter to enter nymph mode, and shift to enter
    egg mode. (when in one of the modes, the row should be lightly
    highlighted

- Visual Style: The app should be friendly with bright colors. The rows
  within the columns should clearly show the corresponding entry key to
  the right of the corresponding photo (each column should have three
  photos corresponding to the three stages of development) The columns
  should be their own bubble / box, with some small separation between
  them. The rows within the columns should also be bubbles, and they
  should mostly fill the column

  - Each row bubble, should contain an image, the name and the key cap
    corresponding. If the bubble gets clicked it should pop up a more
    detailed ID specification that remains within the column.

  - Please attempt to cite the images (if possible).

- At the top there should be a add row button with the ability to add
  your own or select one from one of the rows not currently in use.
